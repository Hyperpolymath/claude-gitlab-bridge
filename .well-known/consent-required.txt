# HTTP 430 Consent Required Protocol
# For Claude GitLab Bridge

## Purpose

This file implements the HTTP 430 "Consent Required" protocol, indicating
that certain operations require explicit user consent before proceeding.

## Scope

The following operations require explicit user consent:

### Data Collection
- Collection of repository data
- Collection of issue/MR content
- Collection of user activity data
- Storage of API tokens
- Caching of GitLab data

### API Operations
- Accessing private repositories
- Reading confidential issues
- Accessing merge request data
- Reading user profile information
- Making changes to GitLab resources

### AI Processing
- Sending code to Claude API
- Processing repository content
- Analyzing issue/MR data
- Generating AI responses
- Storing conversation history

## Consent Requirements

Users must explicitly consent to:

1. **Data Access**: Reading GitLab repository and issue data
2. **AI Processing**: Sending data to Anthropic Claude API
3. **Data Storage**: Temporary caching of API responses
4. **API Usage**: Making requests on behalf of the user

## Consent Mechanism

Consent is obtained through:

1. Installation: User consent during initial setup
2. Configuration: Explicit permission for each repository
3. Runtime: Per-operation consent for sensitive actions
4. Revocation: User can revoke consent at any time

## No Consent = No Operation

Without explicit user consent:
- No data is collected
- No API calls are made
- No information is processed
- HTTP 430 status returned

## Withdrawal of Consent

Users can withdraw consent by:
- Uninstalling the application
- Revoking API tokens
- Disabling repository access
- Contacting: privacy@hyperpolymath.dev
