# Security Policy

If you discover a security vulnerability in Claude GitLab Bridge, please report it privately to our security team.

## Reporting

Please report security vulnerabilities by:

1. Email: security@hyperpolymath.dev
2. GitHub Security Advisory: https://github.com/hyperpolymath/claude-gitlab-bridge/security/advisories/new

## What to Include

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Any proof-of-concept code
* Suggested remediation (optional)

## Response Time

* Acknowledgment: Within 24 hours
* Initial assessment: Within 72 hours
* Status updates: Weekly
* Resolution target: 30-90 days depending on severity

## Disclosure Policy

We follow coordinated disclosure:

1. Report received and acknowledged
2. Issue verified and assessed
3. Fix developed and tested
4. Fix released
5. Public disclosure 30 days after fix (or by agreement)

## Scope

In scope:
* Claude GitLab Bridge application
* GitLab integration components
* Claude API integration
* Authentication and authorization
* Data handling and storage

Out of scope:
* Third-party services (GitLab, Anthropic)
* Social engineering
* Denial of service attacks
* Issues in dependencies (report to maintainers)

## Recognition

We appreciate security researchers and offer:

* Acknowledgment in SECURITY.md
* Credit in release notes (if desired)
* Public recognition (with permission)

Thank you for helping keep Claude GitLab Bridge secure!
